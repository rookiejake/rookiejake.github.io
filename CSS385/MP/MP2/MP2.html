Objective

Now that you have some sense of how the tool and the environment work, let's look at how we can make objects behave as though they have some purpose. In this assignment, you want to: 

  • Practice working and programming simple Finite State Machines.
  • Control the interaction of game objects via simple collision detection.
  • Keep basic statistics of game state.

This assignment is beginning to look more like a typical assignment building towards something potentially useful.

Approach

Please refer to my attempt at the solution (you can either download, unzip, and run (on a PC), or click on this link to run in your browser).  You are welcome to use the textures from my solution, here is the zip file containing all the textures.

With a world height of 200 units (size of the camera should be set to 100), there are four types of objects.

Hero

The hero is a little similar to previous assignment, however, the hero does not travel.

  • It is about 5x5 in size, if your texture image is not square, your hero may not appear square, that is ok.
  • Direction of hero facing is controlled by the AD keys
  • The hero does not move, instead, the hero's position is control by the current mouse position. This is as if the hero is stick on the mouse pointer. This is not to support game play, rather, this makes it simpler for testing (hint, during development, you may alter some game objects behavior for testing purposes).
  • The hero does not interact with the world bound.

Eggs Spawned by the Hero

Once again, space bar spawns eggs at the current hero position.

  • The inter-egg spawning cool down is 0.2 seconds (one egg can be spawned every 0.2 seconds)
  • Spawned egg travels in the current facing direction of the hero at the speed of 100 units per second.
  • An Egg expires when any of the following is true
    • When it is outside the bounds of the world
    • When it collides with an enemy plane, or a way point object
  • Note,
    • 200 unit total height, 100 unit speed, means, an egg will travel across the entire height in about 2 seconds, and,
    • When the hero is located at the bottom of the game window attempting to spawn eggs pointing directly upwards, since an egg will travel and touch the bound in about 2 seconds, with a spawn rate of 0.2 seconds, there can be the most around 10 eggs on the screen at any time in this case.

Way Point System

There are six way point objects in the system, names A to F.

        Their initial locations are: A:(-70, 70), B(70, -70), C(30, 0), D(-70, -70), E(70, 70), F(-30, 0)

        When a way point object collides with an egg, it loses 25% of its opacity

o   This can be implemented by decrease the alpha channel of the SpriteRenderer color by 0.25

        On the forth collision with an egg, a way point object will disappear and re-position itself in a new point that is randomly located at + or - 15 units in both X and Y from the initial position of the way point. Note, this is from the initial position and NOT the current position.

Enemy Planes

Enemy planes operates through the following states. For simplicity, the rotation, enlarge, and shrink states have identical parameters as that of Week 2 Example 5. When traveling, enemy moves at a speed of 20 units per second.

Here are the enemy states:

        Patrol State: This is the default state

o   Under the user control (refer to Game State), a plane sequence through the way points in two one of two ways

  Sequentially from A to F then back to A again

  Randomly from one to the next random way point

o   Target way point, at all times a plane is traveling towards a target way point. It must turn gradually towards the target way point at all times at a rate of 0.03/60. [call the PointAtPosition() function with this rate at each update].

o   When a Target way point is re-positioned (because it has received 4 shots of the egg), the plane will move towards the new position

o   When a plane is within 25 units of the current target way point, it should request and set the next way point to be its new target and begin turning.

o   A collision with the hero sends the plane into CW CCW rotation.

        CCW Rotation

o   The plane turns into red color

o   Please refer to the above specified parameters for the CCW rotation.

o   At the end of the CCW rotation, the plane goes into CW rotation.

        CW Rotation

o   Please refer to the above specified parameters for the CW rotation

o   At the end of the CW rotation, the plane goes into Chase state

        Chase State

o   The plane constantly faces the hero in chase mode, all turnings are immediate

o   As long as the hero is within 40 units of the plane, the plane continues to chase after the hero

o   If the plane collides with the hero during the Chase state

  The plane gets destroyed

  The hero hit (refer to the Game State) is incremented by one

o   If the plane is further than 40 units away from the hero, it transits into Enlarge state

        Enlarge State

o   Please refer to the above specified parameters for the Enlarge State

o   At the end of the Enlarge state, the plane goes into Shrink state

        Shrink State

o   Please refer to the above specified parameters for the Shrink State

o   At the end of the Shrink state, the plane color is changed back to white, and the plane goes back into the Patrol State

        Stunned State

o   At any point, if a plane is hit by an egg, it is sent into Stunned State

o   In Stunned state, the plane texture changed (into something of your choice), and continues to rotate counter-clock wise at a rate of the CCW Rotation State.

o   A hit by an egg of the plane in the Stunned state sends it to an Egg State

        Egg State

o   A plane in Egg State is changed into an egg

o   It is stationary, it is useless

o   A hit by an egg at this stage destroys the plane

Game State

You should keep track of the entire game state.

        H-Key toggles the visibility of all the way point (but the enemies continue to travel among the way points)

        J-Key toggles the planes sequence through the way points, either in A to F sequence, or, in random order

        Q-key quits the game

        At all points, you must echo (print out) the game state

o   Current way point sequence mode

o   Hero status: how many times is it hit by the plane in Chase state

o   Current number of Eggs on screen

o   Plane count: total number of planes on screen (this number is kept at 25) where a new plane must be spawned (at a random position within the world) when one is destroyed

o   Number of planes destroyed by the hero
